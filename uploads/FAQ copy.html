I'm xxxxxxxxxxxxxxxxxxxxxxx using Xcode and I get a "duplicate declaration" error on the line that declares struct Point in Ordered_container_demo2.c. What did I do wrong?

You didn't do anything wrong; what Apple did with this newer version of Xcode is provide a new "module" facility for Objective-C, made it apply to C, and enabled it by default. It drags in the ancient declaration of a Mac OS GUI Point and that causes a collision with your declaration. To fix this, and avoid similar fiascos with our "plain vanilla" code, do the following:


Go into Build Settings (the blue icon at the top of the file list.

Scan down for "Apple LLVM 6.1 - Language - Modules"

Set "Enable Modules (C and Objective C)" to "No".

Do a clean - select Product/Clean

Then rebuild with command-B or the Run button.

This was a nasty surprise. Let me know if you encounter similar set-up problems.


Note: While extremely annoying, this sort of problem happens occasionally with IDEs like Xcode and Visual Studio because they are tools built for full-scale application software development, and we are doing very plain old-fashion code in this course. So the first thing you have to do is figure out how to dumb the IDE down. Both Apple and Microsoft make mistakes of this sort from time to time. In return, you get easy project development, smart editors, and great debugging tools.
